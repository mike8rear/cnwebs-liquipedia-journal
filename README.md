# cnwebs-liquipedia-journal

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存放，便于浏览、索引和长期保存。

## 项目简介

本项目是一个轻量级的静态页面归档仓库。其中包含一系列独立构建或收集的 HTML 文档，每个页面均可直接通过浏览器打开查看。仓库不针对特定网站或域名，而是作为通用存档工具，适用于需要以 HTML 形式发布或备份的各类内容。

## 目录结构

```
.
├── index.html          # 首页 / 索引页（可选）
├── pages/              # 存放归档的 HTML 页面
│   ├── example-1.html
│   ├── example-2.html
│   └── ...
├── assets/             # 公共资源（CSS、图片、字体等）
│   ├── style.css
│   └── images/
├── README.md           # 本文件
└── LICENSE             # 许可证（如有）
```

各目录说明：

- `pages/`：主要归档目录，每个 HTML 文件代表一个独立页面。
- `assets/`：存放页面所需的公共资源文件，按需组织。

## 页面归档说明

- 所有 HTML 页面均为纯静态文件，无需后端服务即可运行。
- 页面文件名建议使用英文小写字母和连字符，避免特殊字符。
- 每个页面应尽量保持自包含，若引用外部资源请确保路径正确。
- 归档内容不保证与原始来源完全一致，仅反映归档时刻的快照。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面或修正现有页面。
- 提交前请确保页面无恶意代码、无外部追踪脚本，且内容不违反适用法律。
- 仓库维护者会不定期检查页面可访问性，并更新或移除失效内容。
- 如发现页面显示异常或链接失效，请提交 Issue 反馈。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/cnwebs-liquipedia-journal.git
   ```
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意 HTML 文件。
3. 也可将仓库部署到 GitHub Pages 等静态托管服务，实现在线访问。
